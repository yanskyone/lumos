### 2. 混淆词持久化修复

**问题**: 选择正确答案后，词对下次训练仍会出现

**根因**: 
- 混淆词对是独立数据，不在用户错题库中
- 没有持久化机制追踪已掌握的词对

**解决方案**:
```javascript
// 新增 localStorage key
CONFUSION_MASTERED: 'lumos:vocab:confusion-mastered'

// 新增方法
getConfusionMasteredPairs()           // 获取已掌握词对
markConfusionPairMastered(key)         // 标记词对为已掌握
getConfusionQuestionsForPractice()     // 获取待练习词对（排除已掌握）
generateConfusionPairKey(pair)         // 生成词对唯一标识
resetConfusionMastered()              // 重置所有掌握状态
```

**验证结果**: ✅ 通过

### 3. 产品方向调整

**背景**: 基于诗颖（真实用户）反馈

**反馈内容**:
- "孩子做题目时更喜欢做选择题"
- "不太喜欢大量输入"

**决策**:
| 训练模式 | 状态 | 原因 |
|----------|------|------|

| **词汇闪电战** | ⏸️ 暂留，后续优化 | 输入形式，后续考虑改成选择题 |
| ~~拼写特训营~~ | ❌ 移除 | 输入形式，孩子兴趣低 |

**代码减少**: ~200行

### 4. 离线优先模式

**问题**: 当前依赖云端数据库，本地调试困难

**解决方案**: 自动检测网络状态，离线时使用本地 JSON 数据

```javascript
// 检测网络状态
const isOnline = navigator.onLine;

if (!isOnline) {
  // 离线 → 直接使用本地 JSON 数据
  _initPromise = initLocalMode();
}

// 网络状态监听
window.addEventListener('offline', () => {
  showToast('💾 已切换到离线模式');
});
```

**本地调试方式**:
1. 断网 + 刷新页面
2. 直接打开文件: `file:///.../vocab/index.html`
3. 本地服务器: `python3 -m http.server 8080`
