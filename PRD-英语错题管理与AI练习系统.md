### v3.1 变更详情（版本方向调整）

#### 🔄 产品方向重大调整

基于**真实用户（诗颖）使用反馈**：

> "孩子做题目时更喜欢做选择题，不太喜欢大量输入"

**调整决策**：

| 训练模式 | 状态 | 原因 |
|----------|------|------|
| **混淆词大作战** | ✅ **重点发展** | 选择题形式，孩子有兴趣，愿意持续学习 |
| **词汇闪电战** | ⏸️ 暂保留，后续优化 | 输入形式，当前兴趣一般，后续考虑改成选择题 |
| ~~拼写特训营~~ | ❌ **移除** | 输入形式，孩子兴趣低，不符合当前用户偏好 |
| ~~从零学词~~ | ❌ 暂不开发 | 输入形式，与诗颖学习场景不匹配 |
| ~~错题复盘墙~~ | ❌ 暂不开发 | 需要更多数据积累后再做 |

#### 📊 用户行为洞察

```
诗颖的学习偏好观察：
├─ ✅ 喜欢：选择题、点击操作、低压力
├─ ❌ 不喜欢：大量文字输入、拼写
└─ 🎯 结论：选择题形式更适合当前用户
```

#### 🎯 新的产品定位

> **Lumos Vocab = 混淆词选择题训练工具**  
> 核心价值：帮孩子通过"区分易混词"建立词汇辨析能力

## 🌟 v2.1 关键设计（保持不变）

诗颖（真实用户）的核心约束：

> 1. "我希望是**鼓励式**的反馈"——不要被指出错误的感觉
> 2. "我每天大概**10 分钟左右**可以，不要太增加我的压力"

**设计原则**：

| 调整维度 | 设计 |
|---------|------|
| **反馈基调** | 鼓励式，看见进步，不指出错误 |
| **训练时长** | **硬上限 10 分钟**（到点自动结束） |
| **整体语调** | 朋友型陪练（"我们一起"） |

## 🔄 训练流程

### 混淆词大作战（核心功能）
```
1. 获取待练习词对（排除已掌握的）
   └─ getConfusionQuestionsForPractice()
2. 随机显示一个词的中文
3. 用户从两个选项中选择正确单词
4. 答对 → markConfusionPairMastered(pairKey)
         → 词对持久化，下次不再出现
5. 答错 → 继续出现，下次仍可练习
```
**特点**：选择题形式，交互简单，符合孩子偏好

### 词汇闪电战（后续优化）
```
1. 获取待复习词（status !== 'mastered'）
2. 显示中文 + 音标
3. 用户输入英文（当前）
   └─ 后续考虑：改为选择题形式
4. 答对 → markCorrect → 下次不再出现
5. 答错 → markIncorrect → 继续出现
```
**优化方向**：考虑将输入题改为选择题，降低输入负担

## 🏗️ 架构优化考虑

### 当前问题

1. **混淆词数据与错题数据分离**
   - 混淆词对是预设的（`CONFUSION_PAIRS`），不在用户错题库中
   - 用户无法自定义添加混淆词对

2. **词汇闪电战与混淆词体验不一致**
   - 闪电战是输入题，混淆词是选择题
   - 后续统一改为选择题

3. **代码模块化程度不够**
   - app.js 中训练相关代码混杂
   - 考虑抽取 training 模块

### 优化方案

#### 方案 A：统一选择题化

```
将所有训练模式改为选择题形式：

┌─────────────────────────────────────┐
│  请选择正确的单词：                   │
│                                     │
│  星期三                              │
│                                     │
│  [ A. wensday ]  [ B. Wednesday ]   │
│                                     │
└─────────────────────────────────────┘
```

**优点**：
- 体验一致
- 降低输入负担
- 代码复用性高

**缺点**：
- 无法训练拼写能力

#### 方案 B：混淆词数据源扩展

```
当前：CONFUSION_PAIRS（预设 32 对）
      ↓ 从用户错题中自动识别

未来：
├─ 预设词对（32对）
├─ 从用户错题自动提取（基于 errorNote）
└─ 用户手动添加
```

**实现**：
```javascript
// 从用户错题中识别混淆词
function extractConfusionPairsFromErrors(errors) {
  // 识别主/宾格混淆（he/she/him/her）
  // 识别名/动混淆（advice/advise）
  // 识别拼写相似词
}
```

#### 方案 C：模块化重构

```
app.js (850行)
    ↓ 抽取

components/
├─ confusion-trainer.js   # 混淆词训练逻辑
├─ flash-war-trainer.js   # 词汇闪电战逻辑
└─ feedback.js            # 反馈语料库

modules/
└─ confusions.js          # 混淆词对数据管理
```

### 推荐优先级

| 优先级 | 优化项 | 理由 |
|--------|--------|------|
| 1 | 移除拼写特训营代码 | 减少代码复杂度 |
| 2 | 验证混淆词持久化 | 核心功能修复 |
| 3 | 增加更多混淆词对 | 丰富内容 |
| 4 | 词汇闪电战选择题化 | 体验一致性 |
| 5 | 模块化重构 | 长期可维护性 |
