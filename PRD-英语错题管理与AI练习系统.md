## 🌟 v2.1 关键调整（基于诗颖的反馈）

诗颖（真实用户）说了两件关键的事，我们马上调整设计：

> 1. "我希望是**鼓励式**的反馈"——不要被指出错误的感觉
> 2. "我每天大概**10 分钟左右**可以，不要太增加我的压力"——暑假作业已经不少了

**这两点决定了 5 个核心设计调整**：

| 调整维度 | 旧设计（v2.0） | 新设计（v2.1） |
|---------|--------------|--------------|
| **反馈基调** | "你漏了字母 d"（指出问题） | "加油！这次更接近了"（看见进步） |
| **错答时** | 高亮缺失字母 | 用笑脸/星星 + "没关系，再试一次" |
| **训练时长** | 默认不限制 | **硬上限 10 分钟**（到点自动结束） |
| **每日任务量** | 推荐 1-2 个训练模式 | 默认 1 个，**让她选"再来一组"才是 2 个** |
| **整体语调** | 工具型提示 | 朋友型陪练（"我们一起" 而不是 "你必须"） |

**产品价值观更新**：

> **Lumos 不是"老师"，是"陪练小伙伴"**  
> 不批评、不排名、不焦虑；只看见她已经做到的，鼓励她继续做。

## 📋 版本变更说明

**v1.0 → v2.0 主要变化**：

1. **产品定位调整**：从"错题录入工具"→**"错题智能训练系统"**
   - 录入是入口，训练是核心
   - 主用户仍是诗颖，但**你（爸爸）是"数据输入员"**

2. **线下 ↔ 线上闭环打通**：
   - 线下：诗颖用"练习版"默写巩固
   - 线上：Lumos 提供"针对性训练"和"进步可视化"
   - 两者各司其职，互相补强

3. **训练模式重设计**：5 个专项训练模式，对应 5 大类错因

4. **数据模型扩展**：支持多批次错题叠加（八年级 + 词组 + 动词）

5. **用户故事重写**：原 P0 全部重写

**v2.0 → v2.1 主要变化**（本次迭代）：

6. **反馈基调调整**：纯鼓励式，不指出错误
7. **训练时长硬上限**：10 分钟自动结束
8. **默认任务量调整**：1 个模式（不是 1-2 个）
9. **整体语调调整**：从"工具"变"陪练小伙伴"
10. **风险预防**：加"压力预防"机制

## 1. 问题陈述

### 1.1 核心问题

杨诗颖（准初三学生）英语学习中最薄弱的是**词汇**。我们已经通过她的231道真实错题（来自七年级上册 Unit 1-6）验证了这一点：

- **58.4%** 的错题是"空白未作答"——不是拼错，是根本记不住那个词
- **22.1%** 是拼写错误（漏字母、多字母、字母顺序、字母替换等）
- 两者合计 **80.5%** 的错题是词汇问题

她真正需要的不是"错题本整理工具"，而是**一个帮她系统性地攻克词汇薄弱点的词汇训练系统**。

### 1.2 数据驱动的需求证明

```
分布：七上 Unit 1-6，涵盖 17 个小节（Expl/Dev/Prep/WU）
类型分布：
  未作答（记忆空缺）    135 题 (58.4%)  ← 最大问题
  拼写错误              51 题 (22.1%)  ← 第二大问题
    其中：漏字母 14 / 完全写错 12 / 元音替代 8 / 多字母 7 / 英美差异 4 / 首尾字母 3 / 字母顺序 3
  词义混淆              10 题  (4.3%)
  语法错误               8 题  (3.5%)
  格式规范               4 题  (1.7%)
  其他                  23 题 (10.0%)
```

### 1.3 真实使用场景（关键洞察！）

**诗颖的实际学习闭环是 OFFLINE + ONLINE 混合的**：

```
线下环节（每天 10-15 分钟）              线上环节（每天 5-10 分钟）
─────────────────────────               ──────────────────────
1. 诗颖用"练习版"纸/Excel 默写          3. 诗颖用 Lumos 训练 5 分钟
   （基于爸爸今天给的"待复习词"）         - 完成每天的"今日训练"
                                            
                                         4. 训练结果自动同步到错题库
                                            
2. 完成后给爸爸批改                     5. 爸爸周末查看 Lumos 战报
   - 答错的标"❌"                          - 看女儿本周进步情况
   - 答对的标"✅"                          - 决定下周训练重点
                                            
                                         6. 爸爸根据战报，给下周的
                                            "练习版"准备 20-30 个词
```

**核心洞察**：
- 错题录入不会发生在 Lumos 内部（孩子不用录入）
- 错题录入是**你（爸爸）的责任**——通过批量导入完成
- 训练本身是**线上的 Lumos** + **线下的默写**两套并行
- 系统的核心价值是**"智能训练"**而不是"数据录入"

### 1.4 不解决的代价

| 维度 | 影响 |
|------|------|
| **中考提分** | 中考英语词汇约占 40% 基础分，21 个单词就扣 1 分（按中考 0.5 分/词估算） |
| **滚雪球效应** | 七年级词汇没掌握，八年级和九年级的句子和阅读会更吃力 |
| **学习信心** | 反复学不会的单词积累到一定量会产生"我就是记不住英语单词"的自我否定 |
| **家长效率** | 买单词书、默写本、网上找练习——全部碎片化，无法追踪进步 |
| **训练无反馈** | 当前线下默写一遍就过，没有"我会了"或"还差 1 个字母"的精细反馈 |

## 2. 目标

### 2.1 用户目标

#### 给诗颖的

> **核心原则：诗颖说"我想要鼓励式反馈"和"10 分钟左右"——这两个约束**  
> **决定了我们所有的设计都要围绕"低压力、高成就感"展开。**

| # | 目标 | 衡量标准 |
|---|------|---------|
| **G1** | **每天 10 分钟左右**做训练，不增加额外压力 | 单次训练 **10 分钟硬上限**，到点自动结束；默认 1 个模式 |
| **G2** | **鼓励式反馈**，让我看到进步，不指出错误 | 错答时不显示"哪里错了"，而是"我们下次一起加油"；每完成 1 个词都有一句鼓励 |
| **G3** | 训练有趣、不像在"写作业"，有游戏感 | 5 个模式差异化（限时/闯关/对比/拼词/周报），有视觉反馈（星星/笑脸） |
| **G4** | 真正记住的词越来越多（通过线下默写验证） | 线下默写正确率从 30% 提升到 60% |
| **G5** | 看到自己的进步轨迹 | 每周看一次"复盘墙"，**只展示"做到了什么"，不展示"还没做到"** |

#### 给 Lumos 的

| # | 目标 | 衡量标准 |
|---|------|---------|
| P1 | 验证"智能训练"模式有效，诗颖愿意用 | 连续 2 周每日训练启动率 ≥ 60% |
| P2 | 错因 → 训练形式 的匹配能产生效果 | 错题 → 已掌握 转化率 ≥ 20%（首月） |
| P3 | 数据模型支持多批次叠加，框架可扩展 | 至少支持 3 批次数据合并（单词 + 词组 + 动词） |

#### 给你（爸爸）的

| # | 目标 | 衡量标准 |
|---|------|---------|
| P4 | 错题录入简单（Excel 导入/粘贴），不要花太多时间 | 单次批量导入 ≤ 5 分钟 |
| P5 | 能看到女儿的进步轨迹，不需要每天干预 | 每周看一次战报，决定下周训练重点 |

## 3. 非目标（Non-Goals）

| 非目标 | 原因 |
|--------|------|
| ❌ 不让孩子在 Lumos 内手动录入错题 | 实际流程是她线下默写，你整理后批量导入 |
| ❌ 不做语法练习（时态/从句/语态） | 实际数据显示仅 3.5% 的错题是语法问题 |
| ❌ 不做对话/情景交际练习 | 错题全部是词汇，与 Unit 7 的情景场景不匹配 |
| ❌ 不做拍照/扫描 OCR 录入 | 初期你手工整理，OCR 工程量大 |
| ❌ 不做数学/物理等其他科目扩展 | 先跑通英语词汇训练的完整闭环 |
| ❌ 不做多用户/多设备同步 | 单用户场景，localStorage 够用 |
| ❌ 不做作文/口语批改 | 范围太大，先聚焦词汇 |
| ❌ 不让 Lumos 取代线下默写 | Lumos 是补充，不是替代——书写的肌肉记忆必须线下完成 |

#### 模块 B：5 个专项训练模式（核心价值）

| ID        | 模式名        | 用户故事                                                   | 验收标准                                                                                                          |
| --------- | ---------- | ------------------------------------------------------ | ------------------------------------------------------------------------------------------------------------- |
| **US-B1** | **拼写特训营**  | 作为诗颖，我希望在拼写错误的词上做针对性训练，能看到"这个字母我总是漏"                   | [ ] 闯关制：每关 5-10 个错词<br>[ ] 模式：看音标 + 中文 → 写英文<br>[ ] 拼错时高亮缺失字母（"← 这里你漏了"）<br>[ ] 拼对 ≥ 80% 进入下一关                |

| **US-B4** | **从零学词**   | 作为诗颖，我希望在"完全写错"的词上被一步步引导拼写，把这个词的拼写"建起来"                | [ ] 模式：拆音节逐字母引导（co-ll-ec-t）<br>[ ] 每个易错字母高亮显示<br>[ ] 关联她已掌握的双写词（"和 exercise 一样的双写"）<br>[ ] 全部拼对 → 词"长出来"，有完成感 |
| **US-B5** | **错题复盘墙**  | 作为诗颖，我希望每周看到自己"掌握了哪些、进步了多少"，而不是只有错误                    | [ ] 每周一推送"上周战报"<br>[ ] 展示：整体掌握率、已掌握 N 个、进步最多的 3 个词<br>[ ] 展示"下周挑战"：3 个最该再练的词<br>[ ] 数字可视化（条形图/饼图）             |

| ID | 用户故事 | 验收标准 |

| **US-C1** | **作为爸爸，我希望在 Lumos 看到"建议诗颖这周线下默写哪些词"** | [ ] 系统基于错题状态推荐：未答对 2 次以上的 + 超过 3 天没练的<br>[ ] 一键导出 20-30 个词为"本周默写清单"<br>[ ] 导出版本类似"练习版.xlsx"的格式（音标 + 中文 + 留空） |

## 5. 需求规格

### 5.1 数据模型（基于真实验证 + 支持多批次）

#### 词汇错题记录（VocabErrorRecord）

```json
{
  "id": "ve-7A-U2-wednesday-001",
  "word": "Wednesday",
  "phonetic": "ˈwenzdeɪ",
  "meaning": "星期三",
  "meaningPartOfSpeech": "n.",          // 自动从释义解析
  "wrongAnswer": "wesneday",            // 她的错误版本
  "errorCategory": "spelling_missing",  // 错因大类
  "errorSubType": "missing_letter_d",   // 错因子类
  "errorNote": "missed letter 'd'",     // 具体描述
  "confusedWith": null,                 // 如果是混淆类，填混淆的词
  "memoryTip": "Wed-nes-day 拆三音节",   // 来自你的"提示"列
  "exampleSentence": null,              // 词典或将来 LLM 补充
  "unitSource": "7A-U2-WU",            // 来源单元
  "batchId": "batch-2026-07-11",       // 批次号（支持叠加）
  "isHistoryImport": true,              // 是否从历史数据导入
  "status": "pending",                  // pending | practicing | mastered
  "masteryLevel": 0,                    // 0-100，间隔重复算法使用
  "nextReviewDate": "2026-07-12",       // 下次复习时间
  "createdAt": "2026-07-11T15:30:00Z",
  "lastPracticedAt": null,
  "lastOfflineTestAt": null,            // 最后一次线下默写时间
  "practiceHistory": [],
  "offlineTestHistory": []
}
```

### 5.2.1 v2.1 鼓励式设计原则（核心价值观）

> **这是 v2.1 最重要的设计原则，决定了所有交互的"语调"**

#### 4 个核心原则

| 原则 | 含义 | 例子 |
|------|------|------|
| **1. 看见做到，不看没做到** | 系统只展示"已经做到的事"，不展示"还没做到" | ❌ 错题 89 个未掌握 / ✅ 已认识 12 个新词 |

| **4. 词是朋友，错题是礼物** | 词汇表不是"待完成任务"，是"交朋友的过程" | ❌ "错题 231 个" / ✅ "我们一起认识 231 个新朋友" |

```javascript
  }
  
  // 混淆类
  if (byCategory.confusion && byCategory.confusion.length >= 1) {
    return ['confusion_battle'];
  }
  
  // 默认
  return ['flash_war'];
}
```

### 6.1 北极星指标

| 指标 | 说明 | 预警线 |
|------|------|--------|
| **训练时长** | **每次训练实际耗时** | **> 10 分钟 → 立即停止（保护机制）** |

## 9. 风险与缓解

| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|---------|
| **诗颖感到压力（暑假作业已经不少）** | **高** | **高** | **10 分钟硬上限** + 默认 1 个模式 + 不显示"还有 N 个没做" + 鼓励式反馈 |
| 诗颖觉得训练"还是抄单词"，没有新鲜感 | 中 | 高 | 5 个模式差异化（限时/闯关/对比/拼词/周报），用游戏化设计 |
| 5 个模式太多她记不住 | 中 | 中 | 首页"今日训练"按钮**只展示 1 个推荐模式**，减少决策成本 |
| 错因自动识别不准确（你答案版的"提示"有时不规范） | 中 | 中 | 识别不出的标"待确认"，让你手动选 1 次，后续复用 |
| 线下默写完没及时录入系统 | 高 | 中 | 周一推送"上周默写成绩单"提醒你录入 |
| 训练时长变长后她不愿意做 | 中 | 高 | 5-10 分钟硬性上限；自动保存进度可中断 |
| **诗颖说"被指出错误让我不开心"** | **中** | **高** | **取消所有"指出错误"的反馈**（详见 5.2.1 节） |
| **诗颖不想用 / 觉得是负担** | **中** | **高** | **产品价值观强调"陪练小伙伴"**，不强迫；她随时可以"明天再来" |
| 多批次叠加导致数据混乱 | 低 | 中 | 按 word 去重 + 按 batchId 分组，不会重复 |

### 9.1 压力预防机制（v2.1 专项）

**触发条件 → 系统自动响应**：

| 触发信号 | 系统响应 |
|---------|---------|
| 训练时间 ≥ 10 分钟 | 弹窗 "今天的训练时间到啦~"（不可绕过） |
| 诗颖当天第 2 次打开 | 提示 "你今天已经练过了~明天见！" |
| 错答 3 次 | 模式自动切换到"从零学词"或提供"看答案"选项 |
| 主动点"不练了" | 不显示任何挽留弹窗，**只说"好的，明天见~"** |
| 诗颖说"不想用" | 立刻记录+产品反馈，但**不要求她解释原因** |
| 训练中停顿 ≥ 2 分钟（无操作） | 提示 "需要休息吗？保存进度，下次继续" |

### 9.2 "你真的不喜欢"的应对（最坏情况）

如果 Lumos 上线后诗颖**就是不想用**：

| 阶段 | 应对 |
|------|------|
| 第 1 周 | 不强求，记录原因。爸爸继续用"线下默写"路径 |
| 第 2 周 | 和她聊聊"哪里让你不喜欢"——可能是某个模式的反馈让她不舒服 |
| 第 3 周 | 如果还是不想用 → 暂停 Lumos 训练，**只保留"线下默写 + 错题清单"** |
| 第 4+ 周 | 重新评估。Lumos 应该是"我想用"，不是"我必须用" |

## 附录 A：批量导入数据格式（支持多批次叠加）

### A.1 TSV 格式（最推荐，从 Excel 复制粘贴即可）

```
word\tphonetic\tmeaning\twrongAnswer\terrorNote
Wednesday\tˈwenzdeɪ\t星期三\twesneday\t漏字母 d
colourful\tˈkʌləf(ə)l\t丰富多彩的\tcolorful\t英式美式
worry\tˈwʌri\tv.担心\t\tworry about 词组错
```

### A.2 字段映射

| 字段 | 必填 | 你的源数据 | 备注 |
|------|------|----------|------|
| `word` | ✅ | 单词列 | 主键，按此去重 |
| `phonetic` | ❌ | 音标列 | 可选，没有也能用 |
| `meaning` | ✅ | 释义列 | 必须有 |
| `wrongAnswer` | ❌ | 诗颖写了列 | 留空表示"未填写" |
| `errorNote` | ❌ | 错误分析/提示列 | 用于自动识别错因 |
| `unitSource` | ❌ | 单元 | 用于按章节分组 |

### A.3 多批次叠加示例

- 批次 1（2026-07-11）：七年级上 231 道（已有）
- 批次 2（预计 7 月底）：八年级单词 100+ 道
- 批次 3（8 月）：词组 50+ 道
- 批次 4（8 月）：动词用法 30+ 道

**系统行为**：
- 同一 word 在不同批次出现 → 保留所有记录，标记为 `batchId` 不同
- 累计 `errorCount` 字段（多次错 → 高优先级）
- 看板显示"该词族的所有错题"（词族聚合）
